## 6. Auth details (docs + code)

- **Access token**: JWT Bearer. `docs/03-api.md:38`: "~15 min TTL"; login example `"expiresIn": 900`. **Code default is now 1 hour**: `apps/panel-api/src/config/configuration.ts:126-128` â€” "Access tokens are silently refreshed; a 1h default (was 15m) cuts refresh churn" â†’ `accessTtl: toInt(process.env.JWT_ACCESS_TTL, 3600)`. `apps/panel-api/.env.example:36` still shows `JWT_ACCESS_TTL=900  # seconds (15m)`. Claims: `sub`, `globalRole`, token type; stateless.
- **Refresh token**: opaque, bound to a `Session` row, only SHA-256 stored (`Session.refreshHash`). Default `refreshTtl` 2592000s = **30 days** (configuration.ts:129). `POST /auth/refresh` **rotates** it; "Reuse of an already-rotated token is treated as theft and revokes the session family" (`docs/08-security.md:56-58`); **60s concurrent-refresh grace window** (`auth.service.ts:37`, `REFRESH_ROTATION_GRACE_MS = 60 * 1000`). **"Trust this device"** (`rememberMe` on LoginDto, `auth.dto.ts:115`): sliding **90-day** refresh TTL, re-extended on every refresh (`auth.service.ts:43`, `TRUSTED_REFRESH_TTL_SECONDS = 90 * 24 * 60 * 60`).
- **2FA**: TOTP (seed AES-256-GCM in `User.totpSecretEnc`) + WebAuthn/FIDO2 passkeys (counter checked monotonic) + single-use recovery codes. Login returns an `mfaRequired` challenge (signed short-lived `mfa`-type JWT; default TTL **300s**, `mfaTtl` configuration.ts:135) instead of tokens; MFA is required when the user has TOTP **or any passkey**. Controller routes (`apps/panel-api/src/auth/auth.controller.ts`): `POST /auth/login` (l.161), `POST /auth/refresh` (l.171), `POST /auth/logout` (l.182), `POST /auth/mfa/verify` (l.129), `POST /auth/mfa/totp/enroll|verify` (l.192,198), `POST /auth/mfa/webauthn/login/options|verify` (l.278,289). (docs/03-api.md names `POST /auth/login/totp`; the code path is `/auth/mfa/verify`.) Sessions listable/revocable: `GET /account/sessions`, `DELETE /account/sessions/:id` (account.controller.ts l.169,174).
- **Password**: Argon2id, constant-time verify (no user-enumeration timing); password-reuse history depth 5 (`auth.service.ts:32`). Preflight (`apps/panel-api/src/config/preflight.ts`) refuses prod boot on placeholder/short/identical `JWT_ACCESS_SECRET`/`JWT_REFRESH_SECRET`.
- **API keys**: prefix + `keyHash`, scopes `READ`/`WRITE`/`ADMIN` (+ `STATUS_READ`), optional `allowedIps` CIDR allowlist, `expiresAt`/`revokedAt`/`lastUsedAt` honoured (`docs/03-api.md:55-60`).
